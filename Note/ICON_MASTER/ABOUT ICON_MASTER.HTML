Table Structure

Each icon follows the correct format with the following fields:

  • ICON_NAME: Human-readable name of the icon
  • ICON_CODE: CSS class or code used to render the icon
  • ICON_LIBRARY: FontAwesome, Bootstrap, Material, Feather, Ant Design
  • ICON_CATEGORY: Logical category for filtering and management
  • PREVIEW_HTML: HTML snippet to render the icon directly
  • ACTIVE: 'Y' for active, 'N' for inactive

Instructions for Developers

  1. Include the CSS/JS library of your icons in the project (FontAwesome, Bootstrap, Material, Feather, Ant Design).
  2. Query ICON_MASTER table to fetch icons by ICON_NAME, ICON_LIBRARY, or ICON_CATEGORY.
  3. Use the PREVIEW_HTML directly in your HTML code to render the icon.
  4. Active icons have ACTIVE = 'Y'; skip inactive ones.
  5. New icons can be added using a standard save procedure (like PROC_ICON_SAVE) for consistent management.
  6. You can filter or group icons in your UI using ICON_CATEGORY or ICON_LIBRARY columns.

Tip: This centralized ICON_MASTER table can be reused across multiple projects, ensuring consistent icon usage and easy updates.
